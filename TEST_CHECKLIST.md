# 测试清单

## 环境准备
- [ ] 以管理员身份运行 `GameAccelerator.UI.exe`
- [ ] 关闭其他代理/VPN 软件

## 一、基础功能

### 1.1 启动/停止
- [ ] 点击「一键加速」，状态灯变绿，状态文字显示"加速中"
- [ ] 托盘图标提示变为"游戏加速器 - 加速中"
- [ ] 点击「停止加速」或托盘菜单「停止加速」，状态灯变红

### 1.2 系统代理
- [ ] 开启加速后，打开 Windows 设置 → 网络和 Internet → 代理
- [ ] 确认「使用代理服务器」已开启，地址为 `127.0.0.1:4433`
- [ ] 停止加速后，系统代理恢复原状

### 1.3 托盘功能
- [ ] 最小化窗口，确认隐藏到托盘（非关闭）
- [ ] 双击托盘图标，窗口恢复显示
- [ ] 右键托盘菜单「显示窗口」「开启/停止加速」「退出」均可正常工作
- [ ] 勾选设置中「启动时最小化到托盘」后重启验证

### 1.4 单实例
- [ ] 程序运行时再次双击 exe，弹出"游戏加速器已在运行中"提示

---

## 二、SNI 代理加速

### 2.1 Steam 加速
- [ ] 开启加速后浏览器访问 `https://steamcommunity.com` 能正常打开
- [ ] 访问 `https://store.steampowered.com` 能正常打开
- [ ] Steam 客户端能正常加载商店/社区/工坊
- [ ] 对比开启前后的加载速度

### 2.2 GitHub 加速
- [ ] 开启加速后浏览器访问 `https://github.com` 能正常打开
- [ ] `git clone` 一个仓库，速度正常
- [ ] 下载 Release 文件正常

### 2.3 游戏加速 (SOCKS5)
- [ ] 确认 SOCKS5 代理监听 `127.0.0.1:10808`
- [ ] 在支持 SOCKS5 代理的游戏/工具中配置代理地址
- [ ] 验证游戏连接正常

---

## 三、流量监控

- [ ] 开启加速后浏览网页，仪表盘下载/上传速度有非零数值
- [ ] 切换到「流量监控」页面，折线图有实时数据
- [ ] 总下载/上传、峰值数据正确更新
- [ ] 停止加速后数据不再变化

---

## 四、服务开关

- [ ] 切换到「服务管理」页面
- [ ] 关闭 Steam 加速开关 → Steam 网站不再通过代理
- [ ] 关闭 GitHub 加速开关 → GitHub 网站不再通过代理
- [ ] 关闭系统代理开关 → Windows 代理设置被还原
- [ ] 各开关状态刷新后保持

---

## 五、设置

- [ ] 修改代理端口后保存，下次启动使用新端口
- [ ] 设置的各项在重启后保持

---

## 六、边界情况

- [ ] 快速连续点击「一键加速」不会崩溃或状态错乱
- [ ] 加速中直接关闭窗口，服务正确停止、代理被还原
- [ ] 非管理员运行时，涉及 hosts/TCP 优化的功能给出明确提示
- [ ] 端口被占用时给出提示而非静默失败
